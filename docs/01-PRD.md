# PRD - ai-ppt-generator

> Product Requirements Document

## 1. 产品概述

### 1.1 产品目标
现有 AI PPT 产品普遍偏向"从零到一全 AI 创作"，但**最广泛的实际使用场景是用户已有固定模板，只需要 AI 帮助把整理好的资料填充到模板中**。

本产品解决这个痛点：让 AI 理解你的模板结构，自动把资料整理填充进去，生成符合模板格式的 PPT。

### 1.2 目标用户
- 需要频繁制作PPT的职场人士
- 企业内部有固定品牌模板，每次只换内容的用户
- 已经整理好资料/知识库，只需要排版到模板的用户
- 对PPT风格/格式有固定要求，不想让 AI 天马行空创作的用户

### 1.3 核心价值
- 尊重用户已有模板，AI 做内容搬运工和整理工，不是创造者
- 发挥 AI 所长：结构理解、内容整理、数据可视化建议、配图找图
- 保留用户模板格式，输出就是最终可直接使用的PPT

## 2. 功能需求

### 2.0 核心交互流程
用户提供两个输入 → 系统三步处理 → 输出最终PPT

```
┌─────────────┐   ┌─────────────┐
│  资料/知识库 │   │  PPT模板文件 │
└──────┬──────┘   └──────┬──────┘
       │                 │
       └─────────┬───────┘
                 ▼
┌─────────────────────────────────┐
│  第一步：模板结构分析识别       │
│  - 识别哪里是封面               │
│  - 识别哪里是目录               │
│  - 识别哪里是标题页             │
│  - 识别各正文页的版式           │
│  - 标记出适合放图片的位置       │
│  - 标记出适合放图表/流程图的位置 │
│  - 统计共有多少个占位页         │
└─────────────┬───────────────────┘
              ▼
┌─────────────────────────────────┐
│  第二步：知识库内容整理         │
│  2.1 生成文字大纲骨架           │
│      - 根据资料提炼各页文字    │
│      - 对应到模板的各个占位页  │
│      - 封面/目录/正文正确映射  │
│  2.2 数据可视化建议             │
│      - 自动识别资料中的表格数据 │
│      - 分析用什么图表展示最合适 │
│      - 给出数据结构化定义      │
│  2.3 附图收集                  │
│      - 从用户知识库中提取相关图 │
│      - 知识库没有则联网搜索相关图│
│      - 明确需求找不到的直接AI生成│
│  2.4 示意图生成                │
│      - 概念性、示意性图片直接生成│
└─────────────┬───────────────────┘
              ▼
┌─────────────────────────────────┐
│  第三步：内容模板匹配合并       │
│  - 将第二步整理的内容放到第一步 │
│    识别出的对应位置             │
│  - 保持原有模板的样式、主题、格式│
│  - 调整文字大小适应占位区域     │
│  - 插入图片到指定图片位         │
│  - 生成图表到指定图表位         │
│  - 输出最终的PPT文件            │
└─────────────┬───────────────────┘
              ▼
        ┌─────────────┐
        │ 输出最终PPT │
        └─────────────┘
```

## 3. 非功能性需求
- **性能**：单模板（≤50页）+ 单资料（≤10万字）处理时间 ≤ 2分钟
- **可用性**：模板识别准确率 ≥ 85%，识别错误可人工修正
- **可扩展性**：模块化设计，模板识别、内容整理、配图、合并四个阶段可独立迭代优化
- **兼容性**：输出兼容 PowerPoint 2016+ 和 WPS 最新版本
- **安全性**：用户上传的模板和资料仅本次处理使用，不长期存储（可选项）

## 4. 用户故事

### 用户故事 1
> "我们公司有统一的品牌PPT模板，每次汇报我都要把内容复制粘贴进去，很麻烦。我希望上传我的模板和整理好的讲稿，直接出来一份做好的PPT。"

### 用户故事 2
> "我已经有了完整的项目资料和数据，也定好了模板风格，只需要AI帮我把内容整理放对位置，找好配图就行，不需要它帮我改模板改风格。"

## 5. 用户操作路径

1. **用户上传**：上传PPT模板 + 资料文档
2. **系统分析**：AI自动识别模板结构 → 展示识别结果给用户确认
3. **用户确认**（可选）：如果识别有误，用户可调整纠正
4. **内容整理**：AI分析资料，生成大纲、数据建议、配图
5. **用户确认**（可选）：用户可调整大纲内容分配
6. **合并生成**：AI将内容填充到模板，生成最终PPT
7. **用户下载**：下载生成好的.pptx文件
